## Business Logic Rules (Nightlife Ticketing)

### Region Selection Requirement
**CRITICAL**: Users MUST select a region before browsing venues/events.
- Region selection is a required first step
- Block access to events/venues list until region is selected
- Show region selection modal/screen if no region selected
- Store selected region in state/context and persist across navigation
- Re-fetch data when region changes

```tsx
// Pattern: Check region before rendering content
const [selectedRegion, setSelectedRegion] = useState<string | null>(null);

if (!selectedRegion) {
  return <RegionSelectionModal onSelect={setSelectedRegion} />;
}

// Only show events after region is selected
const events = await fetchEventsByRegion(selectedRegion);
```

### Ticket Types
All ticket types and their age restrictions:
- **Entry 18+** - General admission, age 18+ required
- **Entry 21+** - Age-restricted admission, 21+ required
- **Drink (21+)** - Drink package, 21+ required for alcohol
- **Combo** - Entry + Drink combo package, 21+ required

### Ticket Redemption Rules

**CRITICAL Business Rules**:
1. **Double Confirmation Required** - All ticket redemptions must require two separate confirmation actions (e.g., double tap, confirm button + confirmation modal)
2. **Customers Cannot Redeem Own Tickets** - Customers must NEVER be able to redeem their own tickets. Only staff can redeem tickets.
3. **Visual Feedback** - Show confirmation state clearly before final redemption

**Redemption Flow Pattern**:
```tsx
'use client';

export default function TicketRedemption({ ticket, userRole }: Props) {
  const [confirmState, setConfirmState] = useState<'idle' | 'confirming' | 'redeemed'>('idle');

  // Rule: Customers cannot redeem tickets
  if (userRole === UserRole.CUSTOMER) {
    return <div>Show QR code - Staff will redeem at venue</div>;
  }

  // Rule: Staff requires double confirmation
  const handleRedeem = () => {
    if (confirmState === 'idle') {
      setConfirmState('confirming');
      // Auto-reset after timeout
      setTimeout(() => {
        setConfirmState(prev => prev === 'confirming' ? 'idle' : prev);
      }, 3000);
    } else if (confirmState === 'confirming') {
      // Second confirmation - execute redemption
      // TODO: Call API to mark ticket as redeemed
      setConfirmState('redeemed');
    }
  };

  return (
    <Button
      onClick={handleRedeem}
      disabled={confirmState === 'redeemed'}
    >
      {confirmState === 'idle' && 'Redeem Ticket'}
      {confirmState === 'confirming' && 'Tap Again to Confirm'}
      {confirmState === 'redeemed' && 'Ticket Redeemed'}
    </Button>
  );
}
```

### Role-Based Permissions

**Role Definitions and Operational Permissions**:

#### Customer Role
- Browse events and venues (after region selection)
- Purchase tickets
- View own tickets in wallet
- **CANNOT** redeem own tickets
- **CANNOT** access staff/merchant/admin features

#### Staff Role
- All customer permissions
- Scan and redeem tickets (requires double confirmation)
- View ticket redemption status
- Access staff dashboard
- **CANNOT** create/edit events
- **CANNOT** access merchant analytics

#### Merchant Role
- All customer permissions
- Create and manage own events
- View own event analytics
- Manage ticket inventory
- **CANNOT** redeem tickets (unless also staff)
- **CANNOT** access admin features

#### Admin Role
- All permissions
- Manage users and roles
- Access all analytics
- System-wide configuration

**Frontend UX Enforcement Pattern**:
```tsx
import { useAuth } from '@/contexts/AuthContext';

export default function EventManagementPage() {
  const { user } = useAuth();

  // Frontend enforcement - hide/disable based on role
  if (user?.role !== UserRole.MERCHANT && user?.role !== UserRole.ADMIN) {
    redirect('/'); // Or show "Access Denied"
  }

  return (
    <div>
      {/* Only show to merchant/admin */}
      {user?.role === UserRole.MERCHANT && <MerchantAnalytics />}
      {user?.role === UserRole.ADMIN && <AdminPanel />}
    </div>
  );
}
```

**Role-Based Component Pattern**:
```tsx
// Conditional rendering based on role
{user?.role === UserRole.STAFF && (
  <StaffRedeemButton ticket={ticket} />
)}

{user?.role === UserRole.CUSTOMER && (
  <CustomerQRCode ticket={ticket} />
  // Never show redeem button to customers
)}
```

## UI Interaction Completion

### Completing Pages and Components

When implementing or reviewing a page/component, ensure ALL of the following are present:

#### 1. Loading States
**MUST** include loading indicators for all async operations:
```tsx
// Data fetching loading
const [loading, setLoading] = useState(true);

if (loading) {
  return (
    <div className="flex items-center justify-center p-8">
      <div className="animate-spin rounded-full h-8 w-8 border-b-2 border-primary"></div>
      <span className="ml-3 text-gray-500">Loading...</span>
    </div>
  );
}

// Button loading state
<Button 
  onClick={handleSubmit}
  isLoading={isSubmitting}
  disabled={isSubmitting}
>
  {isSubmitting ? 'Processing...' : 'Submit'}
</Button>
```

#### 2. Empty States
**MUST** show helpful empty states when no data:
```tsx
if (!events || events.length === 0) {
  return (
    <div className="flex flex-col items-center justify-center py-12 px-4 text-center">
      <span className="material-symbols-outlined text-4xl text-gray-400 mb-4">event_busy</span>
      <h3 className="text-lg font-bold text-gray-300 mb-2">No events found</h3>
      <p className="text-sm text-gray-500 mb-6">Try selecting a different region or date.</p>
      <Button onClick={handleRefresh}>Refresh</Button>
    </div>
  );
}
```

#### 3. Error States
**MUST** display errors with retry options:
```tsx
const [error, setError] = useState<string | null>(null);

if (error) {
  return (
    <div className="flex flex-col items-center justify-center py-12 px-4 text-center">
      <span className="material-symbols-outlined text-4xl text-alert-red mb-4">error</span>
      <h3 className="text-lg font-bold text-gray-300 mb-2">Something went wrong</h3>
      <p className="text-sm text-gray-500 mb-6">{error}</p>
      <Button onClick={handleRetry}>Try Again</Button>

  );
}
```

#### 4. Button States and Logic
**ALL buttons MUST have**:
- Real click handlers (no empty functions or `console.log`)
- Disabled states when not applicable
- Loading states during async operations
- Visual feedback (active, hover, disabled styles)

```tsx
// Good: Complete button implementation
<Button
  onClick={async () => {
    setIsSubmitting(true);
    try {
      await handlePurchase();
      router.push('/wallet');

      setError(err.message);
    } finally {

<Button>Click Me</Button> // ❌ No handler at all
```

#### 5. Form Validation
**MUST** implement validation without external libraries:
```tsx
'use client';

export default function ContactForm() {
  const [formData, setFormData] = useState({ email: '', message: '' });
  const [errors, setErrors] = useState<Record<string, string>>({});
  const [isSubmitting, setIsSubmitting] = useState(false);

  const validate = () => {
    const newErrors: Record<string, string> = {};
    
    if (!formData.email) {
      newErrors.email = 'Email is required';
    } else if (!/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(formData.email)) {
      newErrors.email = 'Invalid email format';
    }
    
    if (!formData.message) {
      newErrors.message = 'Message is required';
    } else if (formData.message.length < 10) {
      newErrors.message = 'Message must be at least 10 characters';
    }
    
    setErrors(newErrors);
    return Object.keys(newErrors).length === 0;
  };

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault();
    
    if (!validate()) {
      return;
    }
    
    setIsSubmitting(true);
    try {
      // TODO: Submit form data
      await submitForm(formData);
    } catch (err) {
      setErrors({ submit: 'Failed to send message. Please try again.' });
    } finally {
      setIsSubmitting(false);
    }
  };

  return (
    <form onSubmit={handleSubmit}>
      <div>
        <input
          value={formData.email}
          onChange={(e) => setFormData({ ...formData, email: e.target.value })}
          className={errors.email ? 'border-alert-red' : ''}
        />
        {errors.email && <span className="text-alert-red text-sm">{errors.email}</span>}
      </div>
      {/* More fields... */}
      {errors.submit && <div className="text-alert-red">{errors.submit}</div>}
      <Button type="submit" disabled={isSubmitting} isLoading={isSubmitting}>
        Send Message
      </Button>
    </form>
  );
}
```

### Code Style for Interactions

**Keep explanations minimal** - Focus on code, not comments:
```tsx

// It first validates the form data, then if valid, it sends the data to the server.

  if (!validate()) return;
```
