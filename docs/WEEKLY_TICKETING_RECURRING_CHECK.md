# 管理员票务配置：循环周 vs 单周 检查报告

**检查日期**：2026-02-17  
**范围**：admin-web 发布活动 → 设置票务 → 客户展示逻辑  

---

## 一、当前实现结论

### 1.1 数据库层面：**已是循环/递归**

`rpc_get_or_create_event_week` 行为：

1. 根据 `p_for_date` 计算 `week_start_date`（周一）
2. 若该周已存在 → 直接返回
3. 若该周不存在 → **自动创建**，并**从上一周复制**：
   - `event_week_days` 的 enabled、start_time、end_time、end_next_day
   - `ticket_types_v2`（票种名称、价格等，不复制 Stripe ID）

结论：管理员只需配置**任意一周**，之后每周在首次被请求时，都会自动沿用上一周配置，无需每周重新设置。

---

### 1.2 管理员端：仅编辑「本周」

| 项目 | 现状 |
|------|------|
| API | `GET /api/admin/events-v2/[id]/week` 默认 `date=today`，无 `?date=` 时即为本周 |
| 页面 | 无周选择器，只能编辑「当前周」 |
| 保存 | 保存到 `week_start_date` 对应的那一周 |

管理员无法在后台切到「下周」预览配置，但从实现看，下周首次被访问时会自动从本周复制，逻辑上是循环的。

---

### 1.3 客户展示：只显示「本周」未结束的场次

**API**：`GET /api/public/events-v2/[id]/week?date=YYYY-MM-DD`  
- 默认 `date=today`  
- 只返回**该周**的配置  
- 过滤：`enabled=true` 且 `valid_end_at > now`（只保留未结束的场次）

**前端**：`events-v2/[id]/page.tsx`  
- 使用 `weekConfig.days` 直接渲染  
- 无「跨周」或「下 N 场」逻辑  

**问题**：

1. 只查当前周，不跨周  
2. 若本周已无未来场次（例如周六晚上），会变成「No tickets available」  
3. 不会自动展示下周的场次  
4. 不保证「始终显示 3 个时间」，可能为 0、1、2 或 3+  

---

## 二、期望的展示逻辑（你的需求）

你希望**始终展示 3 个可购时间**，例如：

- 周四周五周六  
- 周五、周六、下周四  
- 周六、下周四、下周五  

本质是：按「下一个可购日期」排序，取前 3 个可购场次，可跨周。

---

## 三、需要调整的地方

### 3.1 客户 Week API 支持跨周

**思路**：当当前周可展示天数 &lt; N（如 3）时，继续请求下一周，合并结果。

可选实现方式：

- 方案 A：新增 API，例如 `GET /api/public/events-v2/[id]/upcoming-days?limit=3`  
  - 内部调用 `rpc_get_or_create_event_week` 获取本周、下周…直到凑齐 N 个未来场次  
- 方案 B：客户端多次调用现有 `/week?date=xxx`，按周递增 date，合并结果后取前 N 个  

### 3.2 管理员端（可选）

- 增加周选择器：可选择「本周 / 下周 / 下下周」  
- 方便验证下周是否按预期复制，以及是否有需要手动微调的情况  

---

## 四、快速自检步骤

1. **验证循环复制**  
   - 管理员设置某活动本周四、五、六开售  
   - 保存  
   - 下周（如 2 月 24 日）第一次有人访问该活动详情  
   - 预期：下周配置应自动存在，且四、五、六已开启（从上周复制）  

2. **验证客户展示**  
   - 本周六晚上访问活动详情  
   - 若当前实现只查本周：应显示「No tickets available」  
   - 若改为跨周逻辑后：应显示下周四、五、六

## 五、小结（已实现 2026-02-17）

| 项目 | 循环/复用 | 说明 |
|------|-----------|------|
| 数据库 | ✅ 是 | 新周自动从上一周复制 |
| 管理员 | ✅ 一次配置 + 周选择器 | 只需设置一次，之后自动延续；已增加周选择器可预览/编辑下周 |
| 客户展示 | ✅ 已改 | 新增 `upcoming-days?limit=3` 跨周 API，始终显示约 3 个可购场次 |

### 实现改动

1. **`GET /api/public/events-v2/[id]/upcoming-days?limit=3`** - 跨周返回接下来 N 个可购场次  
2. **customer-web 活动详情** - 改用 upcoming-days，单次结账仅允许同周票种，跨周选择时提示  
3. **admin-web 周票务配置** - 增加左右箭头切换「上一周 / 下一周」
